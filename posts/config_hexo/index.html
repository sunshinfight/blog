hexo的配置和使用

发表于 2020-04-07

hexo 安装

首先是得安装node.js，去官网下载就好了，安装完后，使用自带的npm包管理安装hexonpm install hero -g。

hexo 使用 github 构建博客站

找到合适的文件夹下，初始化博客npm init blog-hexo等待完成后会在当前目录下生成blog-hexo文件夹。

blog 简单的配置

./_config.yml文件是blog的配置文件。

配置 next 主题

主题随意去选，然后将主题clone到./themes 文件夹下，我选择next主题。

  1. 先把服务启动起来 hexo s ，然后再浏览器中打开，这样就可以一边修改，一边刷新来查看效果是否满意

  2. 修改blog到配置文件./_config.yml，搜索theme，修改为theme: next那么就代表使用了next主题，会使用themes文件夹下next文件夹中的样式进行渲染

  3. 然后next主题也有它自己的配置，配置文件为./themes/next/_config.yml。不同的主题有不同的配置项，next中主要修改的配置项为：

  4. 如果需要部署到 github 的 pages 服务，那么首先是在github上创建新项目，并且配置pages服务。

  5. 安装部署插件npm install hexo-deployer-git --save，为部署添加配置。

  6. 清理hexo clean，构建hexo g，部署hexo d

使用 markdown 写博客

下载typora，自从发现了这个 markdown 编辑器，再也不想用其他的了。

  1. hexo渲染模版中的一些独特的语法

    • 在md文件中使用该标志，代表在这个标志之前的是文章摘要部分，在home中post列表中，文章会根据这个标志展示，之后紧跟阅读更多按钮

    • Front-matter

      每个文章，都有头，这个头为hexo分类渲染提供信息。

  2. hexo 功能扩展和自定义

    • 图片的处理

      可以不使用图床，配合typora的自定义图片路径和hexo插件可实现图片添加。但是觉得还是不太建议使用图片，因为迁移的话可能会有些资源路径不对的麻烦事。

    • 图片可以使用，但是尽量少用吧，所以添加对图标的支持–mermaid

      首先安装插件npm install hexo-filter-mermaid-diagrams

      然后修改themes/next/layout/_partials/footer.swig文件，添加<script src='https://unpkg.com/mermaid@7.1.2/dist/mermaid.min.js'></script>。
